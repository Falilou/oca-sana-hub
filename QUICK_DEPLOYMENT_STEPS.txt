STEP ④ : PUSH CODE TO GITHUB
═════════════════════════════════════

You'll see a page with commands. Run these in PowerShell:

⚠️ REPLACE: YOUR_USERNAME with your actual GitHub username!

──────────────────────────────────────────────────────────────────────────────

git remote add origin https://github.com/YOUR_USERNAME/oca-sana-hub.git

git push -u origin main

──────────────────────────────────────────────────────────────────────────────

When it asks for username/password:
  • Username: Your GitHub username
  • Password: Your GitHub password (or Personal Access Token)

✅ Your code is now on GitHub!

STEP ⑤ : DEPLOY TO VERCEL (The Magic!)
═══════════════════════════════════════════

2. Enter: https://github.com/YOUR_USERNAME/oca-sana-hub.git
3. Click "Continue"

⏳ Wait 1-2 minutes...

✅ Your app is LIVE! 🎉

Vercel gives you a URL like:

❓ TROUBLESHOOTING

Issue: "Authentication failed"
Solution: Check GitHub username/password, or create Personal Access Token
Link: https://github.com/settings/tokens

Issue: "Permission denied (publickey)"
Solution: Generate SSH key and add to GitHub
Link: https://github.com/settings/keys

Issue: Can't push to GitHub
Solution: Verify repository is public (not private)

Issue: Build fails on Vercel
Solution: Check Vercel dashboard logs, usually missing environment variables
